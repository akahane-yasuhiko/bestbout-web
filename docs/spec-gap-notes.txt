BestBout フロント実装と API/統合仕様の差分メモ
仕様との差分を6項目で整理しました。カード必須情報不足、予想詳細未表示、ページング非対応、ランキング未実装、/cardsクエリ齟齬、Blueskyリンク/評価表示欠落といったポイントを簡潔に記載しています

1. カード情報の不足
   - 必須フィールド (`id`, `event`, `fighters`, `date`, `status`) や詳細 (`venue`, `description`, `result`, `stats`) が UI/型へ取り込まれていない。
   - `poster_image` を使わず常に汎用ポスターを表示。

2. 予想データ表示の不足
   - 仕様で定義されている `method`, `round`, `confidence`, `notes`, `source_url`, `raw_ref`, `hit`, `evaluated_at`, `evaluation_notes` 等が型/UIに存在しない。
   - 表示はハンドル・プラットフォーム・投稿日時・勝者のみ。

3. 予想タイムラインのページング未対応
   - API が返す `nextCursor` を無視しており、追加読み込み UI もない。

4. ランキングページ未実装
   - `/rankings/predictors` を呼ばずスタブ表示のまま。仕様で求められる的中率・総予想数・連勝・最終予想日時が出せない。

5. `/cards` クエリパラメータの齟齬
   - フロントが `status=upcoming` を付与しているが、OpenAPI にはそのクエリ定義がない。

6. Bluesky リンク/評価ステータス未表示
   - `source_uri` などリンク関連、`hit` など評価項目が型/UIにないため、仕様にある外部遷移・的中可視化ができない。

→
本来実装したい仕様との差異について了解しました。
ただ、まずはスクレイピング→DB保存→WEB APIでDBから取得→Reactアプリで表示の一通りを、最小構成でよいので動かすことを優先しようと思います。
現状ではWebAPIがモックから取得したデータをReactアプリで表示、までができた段階ですね？
